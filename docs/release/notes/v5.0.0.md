# Hot Cross Buns 2 5.0.0

Unsigned desktop release for macOS, Linux, and Windows.

## Highlights

- Declares Hot Cross Buns 2 as the maintained release line.
- Ships the Electron-first desktop client for Google Tasks and Google Calendar.
- Adds account workspaces, command palette flows, menu bar surfaces, templates, reminders, smart rescheduling, attachments, ICS import/subscriptions, portable exports, MCP/CLI/webhook automation, and local customization.
- Includes image-derived custom themes and Settings controls for custom backgrounds.
- Publishes macOS DMG/ZIP assets, Linux AppImage assets, Windows NSIS installer assets, stable platform aliases, and SHA-256 checksums.

## Install

### macOS

1. Download `Hot-Cross-Buns-2-macOS.dmg`.
2. Download `SHASUMS256.txt`.
3. Verify checksums:

```sh
shasum -a 256 -c SHASUMS256.txt
```

4. Open the DMG and move `Hot Cross Buns 2.app` into `Applications`.
5. If macOS blocks first launch, use `System Settings > Privacy & Security > Open Anyway`.

### Linux AppImage

1. Download `Hot-Cross-Buns-2-linux-x64.AppImage`.
2. Download `SHASUMS256.txt`.
3. Verify checksums:

```sh
sha256sum -c SHASUMS256.txt --ignore-missing
```

4. Make the AppImage executable:

```sh
chmod +x Hot-Cross-Buns-2-linux-x64.AppImage
```

5. Launch it:

```sh
./Hot-Cross-Buns-2-linux-x64.AppImage
```

### Windows NSIS

1. Download `Hot-Cross-Buns-2-windows-x64.exe`.
2. Download `SHASUMS256.txt`.
3. Verify checksums with PowerShell:

```powershell
Get-FileHash .\Hot-Cross-Buns-2-windows-x64.exe -Algorithm SHA256
```

4. Run the NSIS installer.

## Known Issues

- Builds are unsigned. macOS may require Open Anyway, and Windows or browsers may show SmartScreen/download warnings.
- macOS builds are not notarized.
- In-place auto-update is intentionally disabled; Settings > About checks GitHub Releases and opens a manual download path.
- Google sync uses your own Google Cloud Desktop OAuth client.
- Linux local notifications, global shortcuts, tray/status area, open-at-login, and `hotcrossbuns://` deep links are unsupported.
- Linux credential storage requires an OS-backed provider such as GNOME Keyring/libsecret or KWallet; plaintext fallback is refused.
- Windows signing and SmartScreen reputation are not configured for this release.
