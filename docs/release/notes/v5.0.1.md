# Hot Cross Buns 5.0.1

Unsigned desktop release for macOS, Linux, and Windows.

## Highlights

- Publishes the current Hot Cross Buns release line from the canonical `hot-cross-buns` repository.
- Ships the local vault and hoster backend work advertised on the project site.
- Keeps macOS DMG/ZIP, Linux AppImage, Windows NSIS installer assets, stable platform aliases, and SHA-256 checksums aligned with the `v5.0.1` release.

## Install

### macOS

1. Download `Hot-Cross-Buns-macOS.dmg`.
2. Download `SHASUMS256.txt`.
3. Verify checksums:

```sh
shasum -a 256 -c SHASUMS256.txt
```

4. Open the DMG and move `Hot Cross Buns.app` into `Applications`.
5. If macOS blocks first launch, use `System Settings > Privacy & Security > Open Anyway`.

### Linux AppImage

This Linux AppImage is a technical preview.

1. Download `Hot-Cross-Buns-5.0.1-linux-x86_64.AppImage` or the stable alias `Hot-Cross-Buns-linux-x64.AppImage`.
2. Download `SHASUMS256.txt`.
3. Verify checksums:

```sh
sha256sum -c SHASUMS256.txt --ignore-missing
```

4. Make the AppImage executable:

```sh
chmod +x Hot-Cross-Buns-linux-x64.AppImage
```

5. Launch it:

```sh
./Hot-Cross-Buns-linux-x64.AppImage
```

Linux notifications, global shortcuts, tray/status area behavior, and `hotcrossbuns://` deep links are unsupported.

### Windows NSIS

This Windows NSIS installer is a technical preview.

1. Download `Hot-Cross-Buns-5.0.1-windows-x64.exe` or the stable alias `Hot-Cross-Buns-windows-x64.exe`.
2. Download `SHASUMS256.txt`.
3. Verify checksums with PowerShell:

```powershell
Get-FileHash .\Hot-Cross-Buns-windows-x64.exe -Algorithm SHA256
```

4. Run the NSIS installer.

## Known Issues

- Builds are unsigned. macOS may require Open Anyway, and Windows or browsers may show SmartScreen/download warnings.
- macOS builds are not notarized.
- In-place auto-update is intentionally disabled; Settings > About checks GitHub Releases and opens a manual download path.
- Google sync uses your own Google Cloud Desktop OAuth client.
- Linux credential storage requires an OS-backed provider such as GNOME Keyring/libsecret or KWallet; plaintext fallback is refused.
- Windows signing and SmartScreen reputation are not configured for this release.
