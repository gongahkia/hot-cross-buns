Hot Cross Buns

Privacy and trust

What Hot Cross Buns stores, requests, and does not send.

Hot Cross Buns is a Mac app that talks directly to Google Tasks and Google Calendar. There is no separate Hot Cross Buns backend.

Google access

Hot Cross Buns asks Google only for the scopes it needs to read and write your selected task lists and calendars:

  • https://www.googleapis.com/auth/tasks
  • https://www.googleapis.com/auth/calendar

Those scopes are used for sign-in, sync, and edit operations against Google APIs. Hot Cross Buns does not request Drive, Gmail, Contacts, or Photos access.

Where local data lives

By default the app keeps a local cache under ~/Library/Application Support/com.gongahkia.hotcrossbuns.mac/.

  • cache-state.json: account metadata, settings, sync checkpoints, pending offline mutations, and non-event mirrors
  • cache-events.json: calendar event mirrors stored separately so routine saves stay small
  • cache-state.salt: encryption salt when cache encryption is enabled
  • crash.log: local crash note written on certain fatal crashes

The cache is replaceable. Google remains the source of truth for synced tasks and events.

Encryption and credentials

Google OAuth tokens stay in the macOS Keychain. Optional local cache encryption uses AES-256-GCM with a passphrase-derived key. That encryption protects local cache files on your Mac; it does not change what Google stores.

Crash logs and telemetry

Hot Cross Buns does not include a third-party analytics or crash-upload SDK. macOS may still produce system crash reports in your Library logs, and the app may write a local crash note next to the cache to help diagnose launch failures. Those files stay on-device unless you choose to share them.

Notes, tasks, and events

Tasks and calendar events are backed by Google Tasks and Google Calendar. Notes in the app are task-shaped items that use Google Tasks storage, typically by keeping the note body in the Google Tasks notes field and leaving the due date empty.

Permissions

Notifications are optional and used only for device-local reminders. Accessibility permission is only needed if you enable global hotkeys that should work outside the app. You can revoke Google account access from your Google account settings and disconnect Hot Cross Buns from the app’s Settings window.

Release integrity

Each GitHub release publishes a matching SHA-256 checksum file for the DMG. The install script downloads both the DMG and checksum, verifies them locally, and refuses to install if they do not match.

Distribution model

Preview DMGs are currently distributed outside the Mac App Store. They may be unsigned or not notarized, so macOS can require a one-time manual approval under System Settings > Privacy & Security. Hot Cross Buns can check GitHub Releases for a newer DMG, but installation is still a manual replace rather than a silent in-place update.
